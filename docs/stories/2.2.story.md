## Story

**Como** aluno do workshop "Living Lab Azure-Native",
**eu quero** completar a Fase 2 em 6h provisionando product/subscription no APIM Developer compartilhado e configurando policies completas (`rate-limit-by-key`, cache, JWT validation, CORS, transformations) à frente das Functions de F1,
**para que** eu compreenda na prática o papel de um API Gateway profissional, a diferença entre Consumption (limitado) e Developer (full surface), e tenha policies funcionais que evolvem nas próximas fases.

## Acceptance Criteria

1. **AC-1 — Branch criada:** `phase-02-apim` partindo de `phase-01-servicebus-functions`, com workflow `.github/workflows/deploy-phase-02.yml`.

2. **AC-2 — APIM compartilhado já provisionado pelo instrutor** (cenário A do blueprint): 1 instância APIM Developer em `apim-fifa2026-workshop` provisionada PRÉ-evento. Aluno NÃO cria APIM, mas cria seu product + subscription dentro dele.

3. **AC-3 — Product + Subscription do aluno provisionados via Portal:** aluno demonstra navegação Portal APIM → Products → `+ Add` → product `student-<iniciais>` → Subscriptions → gerar key.

4. **AC-4 — API backend registrada apontando para Function F1:** APIM → APIs → `+ Add API` (HTTP) → backend URL = URL da Function App F1 do aluno. Operations: `POST /purchase`, `GET /purchase/{correlationId}`.

5. **AC-5 — Policy `rate-limit-by-key`:** 5 chamadas por minuto por subscription key. Aluno demonstra: 6ª chamada retorna 429.

6. **AC-6 — Policy `cache-lookup` + `cache-store`:** GET `/purchase/{correlationId}` cached por 30s. Aluno demonstra: 2ª chamada idêntica responde em < 50ms vendo header `X-Cache: HIT`.

7. **AC-7 — Policy `validate-jwt` (preparatório):** policy configurada mas em modo `<choose>` desabilitada (vira mandatória em F3 com External ID). Aluno comenta o `<choose>` para mostrar que aceitaria tokens do issuer Entra quando habilitado.

8. **AC-8 — Policy `cors`:** allow-origin restrito ao domínio do frontend Vite (`fifa2026-web.azurewebsites.net`).

9. **AC-9 — Policy `set-header` / `rewrite-uri`:** request header `X-Correlation-ID` propagado para backend; URI rewrite remove prefixo se necessário.

10. **AC-10 — Smoke test ponta-a-ponta:** chamar `https://apim-fifa2026-workshop.azure-api.net/student-<iniciais>/purchase` com subscription key → policy aplica rate-limit-by-key → encaminha para Function F1 → mensagem na queue.

11. **AC-11 — Tracing end-to-end:** APIM "Test" console + App Insights mostram trace correlacionado APIM → Function → Service Bus → Consumer.

12. **AC-12 — Policy library reutilizável:** arquivo `apim/policies/standard-policies.xml` no repo com policies parametrizadas — serve de base para F3+.

13. **AC-13 — 6 artefatos didáticos** em `docs/workshops/phase-02/` (README + PORTAL-GUIDE com Steps APIM + product + subscription + API + Policies + SPEAKER-NOTES + slides + intro-video-script).

14. **AC-14 — Anti-hallucination (Art. IV):** policies rastreáveis à documentação oficial APIM; NÃO inventar policies que não existam.

## Dev Notes

### Arquitetura técnica delta

```
[Browser]
   │ POST /student-<iniciais>/purchase + subscription key
   ▼
[APIM Developer compartilhado: apim-fifa2026-workshop]
   ├─ Product: student-<iniciais>
   ├─ Subscription: key única por aluno
   ├─ Policies: rate-limit-by-key, cache, cors, validate-jwt (placeholder), set-header
   └─ Backend: Function App F1 do aluno (phase-01)
            │
            ▼
        (resto do fluxo F1: Service Bus → Consumer → SQL)
```

### Policy library padrão (esqueleto)

```xml
<policies>
  <inbound>
    <base />
    <rate-limit-by-key calls="5" renewal-period="60" counter-key="@(context.Subscription.Id)" />
    <cors>
      <allowed-origins>
        <origin>https://fifa2026-web.azurewebsites.net</origin>
      </allowed-origins>
      <allowed-methods><method>GET</method><method>POST</method></allowed-methods>
    </cors>
    <choose>
      <when condition="@(false)"> <!-- F3 ativará: @(context.Variables.GetValueOrDefault<bool>("enableJwtValidation", false)) -->
        <validate-jwt header-name="Authorization" require-scheme="Bearer">
          <openid-config url="https://login.microsoftonline.com/{TENANT_ID}/v2.0/.well-known/openid-configuration" />
        </validate-jwt>
      </when>
    </choose>
    <set-header name="X-Correlation-ID" exists-action="skip">
      <value>@(Guid.NewGuid().ToString())</value>
    </set-header>
  </inbound>
  <backend>
    <base />
  </backend>
  <outbound>
    <base />
    <cache-store duration="30" />
  </outbound>
</policies>
```
