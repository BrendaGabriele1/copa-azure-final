## Story

**Como** aluno do workshop "Living Lab Azure-Native",
**eu quero** completar a Fase 5 em 8h expondo 3 tools MCP (`consultar_disponibilidade`, `verificar_ingresso`, `consultar_bracket`) via Function .NET e integrando um componente React de chatbot no frontend conversando com Gemini 2.0 Flash,
**para que** eu compreenda o protocolo MCP, integração LLM em produtos reais, e tenha um chatbot que responde perguntas reais consultando o SQL do FIFA 2026 Tickets — com portabilidade entre LLMs (Gemini → Groq → Mistral via env var).

---

## Acceptance Criteria

1. **AC-1 — Branch criada:** `phase-05-ai-mcp` partindo de `phase-04-orchestration`, workflow `.github/workflows/deploy-phase-05.yml`.

2. **AC-2 — MCP Server implementado em Function .NET dedicada** (separada das Functions de compra) — `Fifa2026.V2.McpServer/` com endpoint `/mcp` JSON-RPC 2.0.

3. **AC-3 — Tool `consultar_disponibilidade(jogo)`:** recebe `matchId` ou nome de seleções (e.g., "Brasil x Argentina"), retorna `{ vipDisponivel, cat1Disponivel, cat2Disponivel, precoVip, precoCat1, precoCat2 }` consultando SQL.

4. **AC-4 — Tool `verificar_ingresso(qr)`:** recebe ID do ingresso, retorna `{ valido, comprador, partida, categoria, dataCompra }`.

5. **AC-5 — Tool `consultar_bracket(rodada)`:** recebe rodada (e.g., "oitavas", "quartas"), retorna jogos + placares + classificados conforme tabela bracket.

6. **AC-6 — Pinning do MCP SDK** documentado em `docs/architecture/ade-002-mcp-pinning.md` (@architect): versão específica, justificativa, plano de upgrade.

7. **AC-7 — Componente React de chatbot** em `Lovable/.../src/components/Chatbot.tsx` com UI mínima (input + histórico + botão enviar).

8. **AC-8 — Integração Gemini 2.0 Flash:** chatbot envia mensagens para Gemini com tools MCP disponíveis; Gemini chama MCP server quando precisa de dados; resposta volta ao usuário.

9. **AC-9 — Portabilidade via env var:** trocar `LLM_PROVIDER=gemini|groq|mistral` no frontend faz chatbot funcionar com qualquer um sem mexer no código. Demonstrar com Groq como plano B.

10. **AC-10 — Smoke tests didáticos** (3 perguntas-canônica):
    - "Tem ingresso para Brasil x Argentina?" → Gemini chama `consultar_disponibilidade` → responde com disponibilidade e preço
    - "Esse QR code aqui ABC123 é válido?" → Gemini chama `verificar_ingresso`
    - "Quem está nas oitavas?" → Gemini chama `consultar_bracket`

11. **AC-11 — Fallback documentado para Gemini cair:** SPEAKER-NOTES incluem cache de respostas + procedimento de troca emergencial para Groq.

12. **AC-12 — 6 artefatos didáticos** em `docs/workshops/phase-05/` incluindo seção bônus "portabilidade entre LLMs".

13. **AC-13 — Anti-hallucination (Art. IV):** spec MCP rastreável a https://modelcontextprotocol.io/. NÃO inventar métodos JSON-RPC que não existam no spec.

## Dev Notes

### Arquitetura técnica delta

```
[Browser - frontend Vite]
   │ usuário digita "Tem ingresso pra Brasil x Argentina?"
   ▼
[Chatbot.tsx (React)]
   │ useLlmChat('gemini')
   ▼
[Gemini 2.0 Flash API]
   │ recebe pergunta + tools list (MCP)
   │ decide chamar consultar_disponibilidade
   ▼
[MCP Server Function: /mcp JSON-RPC]
   │ tools/call → consultar_disponibilidade(matchId=N)
   │ SQL SELECT em tickets + matches
   ▼
[Gemini]
   │ recebe resultado
   │ gera resposta natural
   ▼
[Chatbot UI]
   │ exibe resposta ao usuário
```

### MCP tools schema (JSON Schema)

```json
{
  "name": "consultar_disponibilidade",
  "description": "Consulta disponibilidade e preços de ingressos para uma partida da Copa 2026",
  "inputSchema": {
    "type": "object",
    "properties": {
      "matchId": { "type": "integer" },
      "matchDescription": { "type": "string", "description": "ex: Brasil x Argentina" }
    },
    "required": []
  }
}
```

(Pattern análogo para `verificar_ingresso` e `consultar_bracket`.)

### LLM providers (env-var switch)

| Provider | Endpoint | Free tier | Cartão |
|---|---|---|---|
| gemini | `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent` | 15 RPM, 1.5K req/d, 1M tok/d | NÃO |
| groq | `https://api.groq.com/openai/v1/chat/completions` | ~30 RPM generoso | NÃO |
| mistral | `https://api.mistral.ai/v1/chat/completions` | Experimentation tier | NÃO |

### Conteúdo bônus: portabilidade

SPEAKER-NOTES inclui demo de 5min: aluno troca `LLM_PROVIDER=gemini` → `groq` → `mistral` no frontend e mesma pergunta produz respostas equivalentes via MCP.

### Testing

**Test approach:**
- Unit: handlers de tools (mock IDbConnection)
- Integration: chamada real ao Gemini com 3 perguntas-canônica
- Smoke: ponta-a-ponta com timeout 10s
