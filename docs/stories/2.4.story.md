## Story

**Como** aluno do workshop "Living Lab Azure-Native",
**eu quero** completar a Fase 4 em 6h subindo n8n self-hosted em Azure Container Apps (Consumption plan, basic auth obrigatório) e desenhando um workflow visual que orquestra ações pós-compra,
**para que** eu compreenda workflow automation low-code, container hosting na Azure, e tenha uma orquestração visual sendo disparada pelo consumer F1.

## Acceptance Criteria

1. **AC-1 — Branch criada:** `phase-04-orchestration` partindo de `phase-03-identity`, workflow `.github/workflows/deploy-phase-04.yml`.

2. **AC-2 — Container Apps environment provisionado via Portal:** Portal → Container Apps → `+ Create environment` → `cae-fifa2026-<iniciais>` (Consumption plan, East US 2).

3. **AC-3 — n8n container app criado via Portal:** image `n8nio/n8n:latest` (pinned a tag stable específica), env vars `N8N_BASIC_AUTH_ACTIVE=true`, `N8N_BASIC_AUTH_USER=admin`, `N8N_BASIC_AUTH_PASSWORD=<gerar>`, `N8N_HOST=<container-app-fqdn>`, `WEBHOOK_URL=https://<container-app-fqdn>`.

4. **AC-4 — Persistência:** Azure Files mountado em `/home/node/.n8n` para preservar workflows entre restarts.

5. **AC-5 — Workflow "post-purchase-notification" desenhado** com 4 nodes:
   - Webhook trigger (recebe correlationId + payload do consumer)
   - Function/Switch (condicional: se VIP → branch A; else → branch B)
   - HTTP Request (mock e-mail via MailHog OU SendGrid free tier OU httpbin.org)
   - Log estruturado

6. **AC-6 — Consumer F1 atualizado para chamar n8n:** após gravar em SQL, PurchaseConsumerFunction faz POST no webhook URL do n8n com `{ correlationId, userId, category, amount }`.

7. **AC-7 — Execuções visíveis no n8n UI:** aluno demonstra histórico de execuções com payload, tempo e status.

8. **AC-8 — Smoke test ponta-a-ponta:** compra v2 → consumer grava → consumer dispara webhook n8n → workflow executa → log + "e-mail" enviado. Tudo visível no n8n + App Insights.

9. **AC-9 — Segurança:** basic auth obrigatório (validar que tentativa sem auth retorna 401); n8n NÃO exposto a internet sem auth.

10. **AC-10 — 6 artefatos didáticos** em `docs/workshops/phase-04/`.

11. **AC-11 — Anti-hallucination (Art. IV):** n8n env vars rastreáveis a docs oficial n8n. NÃO inventar variáveis inexistentes.

## Dev Notes

### Arquitetura técnica delta

```
[Consumer F1]
   │ POST webhook
   ▼
[Azure Container App: n8n]
   ├─ image: n8nio/n8n:<pinned>
   ├─ basic auth: admin / <gerada>
   ├─ persistence: Azure Files /home/node/.n8n
   └─ workflow: post-purchase-notification
        ├─ trigger: webhook /webhook/purchase
        ├─ switch: VIP vs others
        ├─ http: mock e-mail (httpbin.org / mailhog)
        └─ log: console.log estruturado
```

### Env vars n8n essenciais

```
N8N_BASIC_AUTH_ACTIVE=true
N8N_BASIC_AUTH_USER=admin
N8N_BASIC_AUTH_PASSWORD=<gerar — Key Vault ou input>
N8N_HOST=<fqdn>
WEBHOOK_URL=https://<fqdn>
N8N_PROTOCOL=https
N8N_PORT=5678
DB_TYPE=sqlite (default; suficiente para workshop)
GENERIC_TIMEZONE=America/Sao_Paulo
```

### Container Apps tier

- Consumption plan (scale-to-zero quando idle)
- Min replicas: 0
- Max replicas: 2
- CPU: 0.5 vCore
- Memory: 1Gi

### Webhook URL pattern

`https://<container-app-fqdn>/webhook/purchase` (após ativar workflow)

### Testing

**Test approach:**
- Smoke: trigger compra → ver execução no n8n UI dentro de 10s
- Security: curl sem basic auth → 401
- Persistence: redeploy container → workflows preservados (Azure Files OK)
